# 行动优先型决策仪表盘设计

## 目标

把 V2.8.5 决策仪表盘从“数据汇总表”调整为“每日买卖决策工作台”，让使用者在三秒内看清：今天能不能买、优先处理什么、哪些条件阻断执行。

## 设计原则

1. 结论先于证据，动作先于明细。
2. 长期年度配置不能绕过质量准入、买点过滤和证券账户仓位上限。
3. 所有观察标的都保留在买入候选页，并按绿、黄、红、灰排序；红色必须直接显示阻断原因。
4. 数据缺失或过期必须醒目标注，不得伪装为有效信号。
5. 前六张 Sheet 服务日常决策，原始数据、规则和校验统一后移。

## Sheet 阅读路径

### 01_今日决策

首页控制在一屏内，包含：

- 一句话结论：允许复核买入、仅半额复核、只做风控或仅观察；
- 四个状态卡：市场权限、买入候选、减仓复核、现金安全垫；
- 今日动作队列：最多一个买入方向和两个减仓/卖出方向；
- 三条主要阻断原因；
- 行情时间、来源和数据完整性状态。

### 02_今日动作

按优先级展示动作类型、标的、动作预算、核心依据、当前仓位、目标仓位、剩余额度、年度配置进度和状态。风控/减仓动作排在买入复核之前。

### 03_买入候选

展示全部关注标的，排序顺序为绿、黄、红、灰。同一灯号内优先展示质量评分高、买点通过项多、剩余额度充足的标的。

核心列为：灯号、代码、名称、建议、阻断原因、通过项、质量状态、市场权限、当前仓位、目标仓位、剩余额度、年度配置项、年度缺口和数据状态。完整六项证据保留在后部列。

### 04_持仓风险

优先展示已触发提醒、超配、遗留仓和专项减仓纪律。核心列为：风险级别、标的、动作建议、触发原因、当前仓位、目标仓位、盈亏、纪律价格区间和年度配置状态。

### 05_年度配置

展示第一年度目标占比、动态目标金额、当前金额、资金缺口、完成率和执行约束。明确标注“年度资金缺口不是买入信号”。

### 06_组合总览

展示账户与全资产双口径、现金安全垫、当前仓位对目标仓位、年度配置当前金额对动态目标。首页不再堆放逐条提醒，逐条风险移至 `04_持仓风险`。

### 后台明细页

双锚、情绪、质量评分、组合暴露、行情数据、持仓明细、券商快照、观察池、规则、校验和使用说明依次后移。后台页保留完整数据，不承担首页决策表达。

## 视觉层级

- 红色：禁止新增、超配、减仓触发、数据过期；
- 黄色：需人工复核、信号不足、接近上限；
- 绿色：完整门槛通过且存在仓位额度；
- 灰色：仅观察、不适用或数据缺失；
- 深蓝色：标题和稳定结构，不表达方向。

数字列统一使用百分比、千分位和负数红色格式；正文避免过窄列，关键原因允许自动换行。前六张 Sheet 冻结标题行、开启筛选，并按决策优先级预排序。

## 数据流

`watchlist.csv`、`positions.csv`、`decision_inputs.csv` 和 `first_year_allocation.csv` 继续作为唯一输入。`main.py` 先生成底层行情、质量、买点、持仓风险和年度配置，再构建六张前台决策页。盘前报告优先读取前台页，底层明细仅用于补充证据和审计。

## 错误处理

- 行情抓取失败时仍生成工作簿，但首页必须显示“行情未刷新”，买入候选不得判绿；
- 缺少质量评分或完整证据时，候选保留但显示红色阻断；
- 年度配置代码未映射时显示“待选择合格标的”，不自动创建买入动作；
- 任何核心 Sheet 缺失时，校验页标记失败，盘前报告退化为条件式观察。

## 验收条件

1. 前六张 Sheet 顺序与本设计一致。
2. `01_今日决策` 在正常屏幕宽度下可一屏阅读。
3. `03_买入候选` 包含全部关注标的并按绿、黄、红、灰排序。
4. 红色候选必须有非空阻断原因。
5. `04_持仓风险` 的提醒数与 Dashboard/校验结果一致。
6. 年度配置合计、映射金额和资金缺口勾稽通过。
7. 模板与最新生成工作簿使用同一 Sheet 结构。
8. 工作簿无损坏、无公式错误、无外部链接，关键 Sheet 完成渲染检查。
9. 旧文档清理结果符合保留与删除清单。
